# [pj-name] プロジェクト テンプレート構造

このディレクトリは、[pj-name]（プロジェクト名を任意に変更）のディレクトリ構造テンプレートです。
他プロジェクトへの流用や新規プロジェクトの雛形として利用できます。

---
## ディレクトリ構成（例）

- app/[pj-name]/ : バックエンド構造テンプレート
  - migrations/ : DBマイグレーション用SQL
  - scripts/ : 各種スクリプト
  - src/ : バックエンドソース
  - src-client/ : フロントエンドソース
  - tests/ : テストコード
  - package.json : サンプル設定ファイル
- docs/ : ドキュメント雛形
  - セキュリティ/ : セキュリティ設計・対策
  - マーケティング/ : マーケティング資料
  - 整備/ : 運用・改善資料
  - 追加仕様/ : 拡張仕様資料
  - その他主要ドキュメント（アイデア、要件、基本設計など）
- issue/ : 課題管理雛形
- bk/ : バックアップ雛形

---
## 使い方

1. sample ディレクトリごとコピーし、[pj-name] などの名称を自分のプロジェクト名に一括置換してください。
2. 不要なディレクトリやファイルは削除、必要なものは追加してください。
3. 各README.md や package.json などはプロジェクトに合わせて編集してください。
4. docs 配下の構造も必要に応じてカスタマイズしてください。

---
## 各ファイル・READMEのフォーマット例

- 各README.mdには「このディレクトリの目的」「主なファイル例」「記載例」などを明記してください。
- サンプル:
  - app/[pj-name]/README.md
    - このディレクトリはバックエンドの主要構成を格納します。
    - 例: src/（TypeScriptソース）, migrations/（SQLファイル）など
  - docs/README.md
    - プロジェクト全体のドキュメント管理方針や分類ルールを記載
